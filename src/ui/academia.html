📚 Onde mora o conhecimento (a parte que confunde todo mundo)

Antes dos mecanismos: de onde o Orion tira o que sabe. Não é um lugar só — são vários "cadernos", cada um com um dono e um propósito.
🧩 Analogia: pense num assistente com vários cadernos — uma constituição que quase nunca muda, manuais de cada trabalho, um diário pessoal, uma ficha sobre você, um caderno-wiki compartilhado, e uma memória viva que anota tudo que vocês conversam. E há dois leitores desses cadernos: o plugin no editor de código e o Orion no WhatsApp.

🔗 Como tudo se junta
1. O ingest costura tudo
De hora em hora, o vault + memory/*.md + CLAUDE.md global são copiados para dentro do SQLite (confianças 0.80 / 0.85 / 0.90). Assim o conhecimento curado também aparece nas buscas.
2. Dois leitores, mesmos cadernos
O plugin (editor) lê os CLAUDE.md + MEMORY.md sozinho e consulta o SQLite via MCP. O Orion (WhatsApp/web) monta o contexto com perfil + CLAUDE.md + vault + SQLite + skills.
3. Você fala num lugar, lembra no outro
Como ambos leem o mesmo banco e os mesmos arquivos, uma decisão dita no WhatsApp aparece no editor — e vice-versa. Um cérebro, várias portas.

🎯 Como o Orion decide o que lembrar

Quando chega uma mensagem, cada memória candidata recebe um score composto. Os mais altos entram no contexto.
score = 0.45 × BM25_norm (quão lexicalmente parecido) + 0.26 × recência (exp(-0.0001 × idade_seg)) + 0.16 × Jaccard (overlap de tokens) + 0.13 × HRR (similaridade composicional) ───── tudo isso × confiança_ajustada (× decay por categoria) + feedback_boost (votos úteis/inúteis) + salience_bonus (importância × acessos) descarta se confiança_ajustada < 0.15 (MIN_TRUST) diversifica o topo com MMR (λ=0.7) p/ não repetir tema

📉 Nem toda memória esquece na mesma velocidade

Cada categoria tem uma meia-vida diferente — o tempo até a confiança cair pela metade. Decisões mudam rápido; fatos sobre pessoas duram anos.

⚡ O que acontece quando você manda "qual o status do projeto?"

O fluxo por turno (mensagem substantiva). Saudações pegam um atalho leve.
Recebe e classifica
Gateway filtra o dono, transcreve áudio se preciso, e decide: trivial (atalho rápido) ou substantiva (máquina completa).
Comprime a sessão se longa
Se passou de ~72% do contexto, o Haiku resume turnos antigos em seções estruturadas e extrai fatos antes de descartar.
Monta o contexto (6 camadas)
Perfil + CLAUDE.md do projeto + vault global + vault do projeto + memórias relevantes (score híbrido) + skills.
Recupera memórias
BM25 + vetor + recência + Jaccard + HRR, multiplicados pela confiança com decay, diversificados por MMR, com 5% de "loteria" de memórias esquecidas.
Chama o Claude CLI
Com a memória injetada no user message (nunca no system prompt — preservaria o cache). Resume a sessão ou começa fresca se a anterior travou.
Responde e aprende em background
Envia a resposta. Aí dispara (sem bloquear): extração regex+Haiku, background-review, geração de skill, perfil, grafo causal, índice temporal.

🔗 Um cérebro, três portas

WhatsApp, o plugin do Claude Code e os crons escrevem e leem o mesmo memory.db. Por isso o Orion lembra de você em qualquer lugar.

📱 WhatsApp

  • Evolution chama /webhook/evolution
  • runOrion() monta contexto + responde
  • Cada troca vira memória (episodic)
  • Relatórios e perguntas ativas chegam aqui

🔌 Plugin Claude Code

  • MCP orion-memory (stdio)
  • orion_memory_context antes de responder
  • orion_memory_save grava fatos
  • orion_context traz o pacote completo
  • Mesmo banco que o WhatsApp

⏰ Cron

  • Phase 2 (30min) · Phase 3 (6h)
  • Phase 4 (diário) · Phase 5 (semanal)
  • Phase 6 (mensal) · Phase 7 (on-demand)
  • Cura, consolida e aprende sozinho
